中州养老管理系统

项目简介

本项目基于“RuoYi-Vue 若依框架”开发，面向养老院、养老机构实现全流程信息化管理，包含后端管理服务、Vue3 管理前端、微信小程序三端完整代码。



!\[中州养老管理系统整体效果图](./images/系统首页截图.png)



项目模块说明

1\. xian110 后端服务

&#x20;  SpringBoot + MyBatis-Plus 业务后端，提供老人档案、护理等级、床位管理、收费工单、排班、体检等业务接口，集成若依权限控制、数据字典、文件上传、日志管理。

2\. xian110-vue3 管理前端

&#x20;  Vue3 + Element Plus 后台管理页面，实现数据分页、新增编辑、导出、统计图表、角色菜单权限配置。

3\. mp-weixin 微信小程序端

&#x20;  老人家属小程序，提供信息查询、预约探视、缴费、消息通知功能。

4\. zzyl.sql

&#x20;  项目完整数据库初始化脚本，包含业务表、字典、基础菜单、测试数据。



技术栈

后端

Java 17、SpringBoot、MyBatis-Plus、MySQL、Redis、Maven

管理前端

Vue3、Vite、Element Plus、Axios

小程序

原生微信小程序、JS、CSS



开发迭代流程

1\. 新建数据表

根据养老业务梳理字段、关联关系，统一添加若依公共字段，设计数据库表结构，搭建底层数据支撑。

2\. 代码生成

使用若依内置代码生成器，一键生成前后端CRUD基础代码，减少重复编码，统一代码规范。

3\. 逻辑微调

基于生成代码定制养老专属业务逻辑，优化页面交互、表单联动、统计展示，适配机构实际运营场景。

4\. 联调测试

后台配置菜单与角色权限，前后端接口联调，多角色全流程功能测试，校验数据流转与边界场景。

5\. 监控上线

清理敏感密钥配置，提交版本仓库；服务打包部署，线上监控日志与接口性能，稳定后正式交付使用。



环境部署

1\. 执行 zzyl.sql 导入数据库

2\. 修改后端 application-dev.yml 数据库、文件存储配置

3\. 后端 Maven 打包 Jar 运行

4\. 前端 npm install / npm run dev 本地调试，npm run build 打包部署

5\. 小程序导入 mp-weixin 目录，修改接口域名后上传发布



注意事项

1\. 公开仓库已移除完整阿里云密钥，部署时自行配置OSS等第三方密钥；

2\. 生产环境禁止使用 dev 配置，切换 prod 生产配置；

3\. 权限严格区分管理员、护理员、财务、家属角色，保证数据隔离。
